합성 시나리오 시각화 — PoC

resource-task model → 시나리오 합성 UI 후보
홈 ① 합성 캔버스 ② 레이어드 DAG ③ 폐포 오버랩 ④ 분기 토글 ⑤ 시퀀스/간트 통합 IA 참고사례

knowledge/formal/resources/*.yaml 의 자원-task 모델(의존 requires + 옵션 options)을 입력으로, "단위 서비스의 의존성 합성"을 화면으로 보여주는 5가지 UI 아이디어의 동작 샘플입니다. 모든 데이터는 실제 모델에서 build_data.py 로 추출했고, 외부 의존성 없이 브라우저에서 바로 열립니다.

UI 아이디어 샘플

Idea ①

인터랙티브 합성 캔버스

대상 노드를 클릭 → 의존 폐포가 그래프에서 켜지고, 합성될 plan(생성→검증→역순삭제)· dedup·peak quota가 실시간으로. 플래그십.
Idea ②

레이어드 DAG + 스윔레인

위상 깊이 = 생성 순서(가로축), 카테고리 = 스윔레인. teardown 역방향 오버레이. 전체 의존 구조를 한눈에.
Idea ③

폐포 오버랩 (집합 뷰)

여러 타깃의 의존 폐포가 겹치는 공유 인프라(dedup으로 1회만 생성)를 강조 — 합성의 절감 가치를 시각화.
Idea ④

OR-의존 분기 토글

one_of(택일 의존)·count(다중성)을 토글로 바꾸며 폐포 비용(_branch_cost)이 갱신되는 비교 뷰.
Idea ⑤

시퀀스 / 실행 간트

합성 plan을 실행 타임라인으로 — async ready 대기, heavy 소요, create/delete 페이즈를 막대로.
통합 ★

IA 워크스루 (Overview→Plan→Run→Report)

위 아이디어를 하나로 — 같은 그래프 컴포넌트가 4개 탭에서 오버레이만 바뀌며 재사용되는 통합 목업. Plan에서 합성→Run 실시간→Report 결과까지.
참고

레퍼런스 갤러리

Terraform graph·CloudFormation Composer·Airflow·Dagster·n8n 등 같은 구조를 이미 보여주는 도구들 — 설명 + 외부 링크 + UI 목업.
학술/엔지니어 명칭: 이 구조는 ① 의존성 DAG의 위상 합성(빌드시스템/패키지매니저의 dependency resolution) ② precondition–effect 자동계획(STRIPS/HTN) ③ 피처 모델 기반 제품 구성(SPL) ④ 모델 기반 테스트(MBT) 가 겹친 형태입니다. 한 줄로는 "declarative composition over a resource DAG (model-based scenario synthesis)".
