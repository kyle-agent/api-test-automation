④ OR-의존 분기 토글 + 다중성

홈 › one_of / count 의 합성 비용 비교
홈① ②③ ④⑤ 참고

one_of(택일 의존, 예: direct-connect OR transit-gateway)와 count(다중성, 예: vpc-peering = vpc×2)는 합성 시 분기 비용이 달라집니다. 분기를 토글하면 폐포·비용(_branch_cost: 더 적은 생성 + 더 적은 heavy 선호)이 갱신됩니다. 이게 disjunctive dependency / 제약 충족(SAT)에 해당하는 부분입니다.

분기/다중성 노드

one_of 또는 count>1 을 가진 노드만.

폐포 그래프

분기 비용 비교

다중성

비용 = 폐포 생성 수(적을수록 저렴) · heavy 수 · 분기 목록 순서로 tie-break (composer.py _branch_cost). 모델에 one_of/count가 더 들어올수록 이 화면의 가치가 커집니다.
